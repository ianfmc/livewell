# LIVEWELL – Sprints

This folder contains the sprint-by-sprint execution history for the LIVEWELL Nadex project.
Each sprint has **one canonical sprint definition file** and, when warranted, a **separate retrospective**.

This structure keeps intent (what we planned to do) cleanly separated from reflection (what actually happened and why).

---

## How to Read This Folder

1. **Start with the sprint file** (`sprint_X.md`)
   - Sprint goal and scope
   - User stories targeted
   - Definition of Done
   - Planned vs actual dates
   - Backlog and outcomes

2. **If present, read the retrospective** (`retrospectives/sprint_X.md`)
   - What went well
   - What didn’t
   - Adjustments to process, scope, or tooling
   - Inputs into future sprint planning

Not every sprint requires a retrospective. Retrospectives are written when there is meaningful learning to capture.

## Conventions

- **One sprint = one authoritative sprint file**
- Retrospectives are optional, additive, and never rewrite history
- Completed sprint files are not edited except for date corrections
- The project canvas is the source of truth for sprint ordering and status
